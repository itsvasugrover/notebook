# 엔지니어링 노트

**바수 그로버**의 디지털 브레인에 오신 것을 환영합니다.

이곳은 전통적인 문서 사이트가 아닙니다. 복잡한 임베디드 시스템을 구축하는 과정에서 마주한 아키텍처, 결정, 그리고 "가혹한 진실"을 기록하는 살아있는 **엔지니어링 로그**입니다. 임베디드 개발을 취약하고 임시적인 프로세스에서 견고하고 확장 가능하며 안전한 엔지니어링 실천으로 변화시키는 여정을 공유합니다.

## 미션

**펌웨어 엔지니어링**, **DevOps 인프라**, AI 및 IoT와 같은 신기술을 연결하여 임베디드 시스템 엔지니어링을 혁신합니다. 대부분의 임베디드 프로젝트가 실패하는 이유는 코드가 아니라, 취약한 빌드 환경, 자동화 부족, 보안 대책 미흡 때문입니다. 개발자가 신뢰성 높고 고성능의 시스템을 구축할 수 있도록 오픈소스 도구, 프레임워크, 베스트 프랙티스를 만듭니다.

목표:

- 임베디드 Linux 배포판을 위한 자동화된 CI/CD 파이프라인 개발
- 펌웨어 개발에서 보안 우선 접근 방식 구현
- 자원 제약 장치에 머신러닝 기능 통합
- 임베디드 DevOps의 커뮤니티 주도 표준 구축

## 현재 연구실

- **Yocto 빌드 팜:** 여러 아키텍처의 펌웨어를 동시에 빌드 및 테스트할 수 있는 분산형 하이브리드 클라우드 CI 시스템(Jenkins + Docker + QEMU)
- **자동차 시스템:** AUTOSAR Adaptive, QNX, 고성능 ECU 아키텍처 심층 분석. 실시간 OS 및 안전 중심 설계 패턴 포함.
- **홈랩 인프라:** 고급 네트워킹, DNS, 가상화 실험, 개발 환경의 컨테이너 오케스트레이션.
- **IoT 게이트웨이 프로젝트:** 산업용 IoT를 위한 안전하고 확장 가능한 엣지 컴퓨팅 게이트웨이 구축.
- **보안 강화:** 임베디드 장치의 시큐어 부트, 암호화, 취약점 평가 연구 및 구현.
- **엣지 AI:** 마이크로컨트롤러에서의 머신러닝 추론 및 저전력 장치용 신경망 최적화.

## 읽는 방법

- **ADR(아키텍처 결정 기록):** 도구 선택 이유와 대안에 대한 상세 설명.
- **빌드 로그:** 실제 컴파일 실패, 툴체인 문제, 최적화 과제 디버깅.
- **런북:** 인프라의 "비상시 대응" 가이드. 재해 복구 및 유지보수 절차 포함.
- **사례 연구:** 성공 프로젝트의 심층 분석과 실패에서 얻은 교훈.
- **튜토리얼:** 개발 환경 설정 및 복잡한 기능 구현 단계별 가이드.

## 비전 및 향후 계획

앞으로의 목표:

- 임베디드 시스템 개발을 촉진하는 오픈소스 프로젝트에 기여
- 엔지니어 지식 및 도구 공유 커뮤니티 구축
- 임베디드 DevOps의 엔터프라이즈 솔루션 개발
- 임베디드 시스템에서의 양자 컴퓨팅 응용 탐구
- 차세대 임베디드 엔지니어를 위한 교육 자료 제작

## 핵심 원칙

- **신뢰성 우선:** 시스템은 견고하고 장애 허용성이 있어야 함
- **보안 설계:** 기본부터 보안을 통합
- **자동화 중심:** 도구로 수작업 제거
- **지속적 학습:** 신기술을 지속적으로 습득
- **개방적 협력:** 지식 공유 및 커뮤니티 기여

## 설치

### 사전 조건

- Node.js >= 20.0
- npm 또는 bun

### 설정

```bash
# 저장소 클론
git clone https://github.com/itsvasugrover/notebook.git
cd notebook

# 의존성 설치
bun install
```

## 개발

```bash
# 개발 서버 시작
bun run docs:dev
```

사이트는 `http://localhost:8080`에서 확인할 수 있습니다.

## 빌드

```bash
# 프로덕션 빌드
bun run docs:build
```
