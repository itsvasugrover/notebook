# 工程笔记

欢迎来到 **瓦苏·格罗弗** 的数字大脑。

这不是传统的文档网站，而是一个不断更新的 **工程日志**，记录了我在构建复杂嵌入式系统过程中遇到的架构、决策和“残酷真相”。在这里，我分享将嵌入式开发从脆弱、临时的流程转变为健壮、可扩展、安全的工程实践的旅程。

## 使命

通过桥接 **固件工程**、**DevOps 基础设施** 和 AI、物联网等新兴技术，彻底变革嵌入式系统工程。大多数嵌入式项目失败并非因为代码，而是由于构建环境脆弱、缺乏自动化和安全措施不足。我致力于通过开源工具、框架和最佳实践，帮助开发者构建可靠、高性能的系统。

我的目标包括：

- 为嵌入式 Linux 发行版开发自动化 CI/CD 流水线
- 在固件开发中实施安全优先的方法
- 在资源受限设备中集成机器学习能力
- 构建嵌入式 DevOps 的社区标准

## 当前实验室

- **Yocto 构建农场：** 一个分布式混合云 CI 系统（Jenkins + Docker + QEMU），可同时为多架构构建和测试固件。
- **汽车系统：** 深入研究 AUTOSAR Adaptive、QNX 和高性能 ECU 架构，包括实时操作系统和安全关键设计模式。
- **家庭实验室基础设施：** 高级网络、DNS、虚拟化实验和开发环境的容器编排。
- **物联网网关项目：** 构建安全、可扩展的工业物联网边缘计算网关。
- **安全加固：** 研究和实现嵌入式设备的安全启动、加密和漏洞评估。
- **边缘 AI：** 探索微控制器上的机器学习推理，并优化低功耗设备的神经网络。

## 如何阅读

- **ADR（架构决策记录）：** 详细说明为何选择某工具，包括权衡和备选方案。
- **构建日志：** 真实调试编译失败、工具链问题和优化挑战。
- **运行手册：** 基础设施的“应急指南”，包括灾难恢复和维护流程。
- **案例研究：** 深度分析成功项目和失败经验。
- **教程：** 开发环境搭建和复杂功能实现的分步指南。

## 安装

### 前置条件

- Node.js >= 20.0
- npm 或 bun

### 设置

```bash
# 克隆仓库
git clone https://github.com/itsvasugrover/notebook.git
cd notebook

# 安装依赖
bun install
```

## 开发

```bash
# 启动开发服务器
bun run docs:dev
```

网站将运行在 `http://localhost:8080`。

## 构建

```bash
# 生产环境构建
bun run docs:build
```

## 部署

本网站部署于 [https://notes.itsvasugrover.com](https://notes.itsvasugrover.com)。

本地部署测试：

```bash
bun run docs:preview
```
